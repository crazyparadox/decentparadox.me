## December 2025 — Day One
</div>

December 24th. I started Neumann on Christmas Eve because apparently that's the kind of person I am.

The idea had been rattling around for a while — a local AI code editor, nothing cloud-dependent, running entirely on your machine. I'd been using Cursor and it was good, but something about having my code leave my machine for every completion bothered me. I wanted to understand what it would take to build one myself.

The first version was GPUI-based. [GPUI](https://github.com/zed-industries/gpui) is Zed's GPU-accelerated UI framework — native, fast, 120fps rendering, a proper terminal emulator already built in. The pitch was: stay in Rust end-to-end and get a professional UI runtime for free. Zed had proven it could work at scale. I figured I'd ride those coattails.

I built fast. In one day I had a basic sidebar, resizable panels, a file tree, a file viewer, and a main UI view. The bones were there immediately. The first few days were the most satisfying part of the whole month — the project was small enough that every commit felt like a big visible win.

By late December I had a working terminal (briefly), syntax highlighting with virtual scrolling, a scrollable file tree, OIDC authentication against a local server, and theme support. The project was called "luban" at this point — Neumann came later.

The terminal getting removed on Dec 28 is worth noting. It was one of the features I most wanted — a proper embedded terminal in the editor. But the GPUI terminal had rendering issues I didn't understand yet, and I didn't want them blocking the rest of the UI work. Pulled it out with the intention of coming back. (I did come back — in March, via `ghostty-web` over WebSocket rather than GPUI at all.)

The project ended December in reasonable shape: a real file tree, syntax-highlighted file viewer, working auth, resizable panels. Enough to keep going.
